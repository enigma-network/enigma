# Enigma Web — Phase 4 Design Spec

**Datum:** 2026-05-07
**Status:** Approved
**Scope:** Provider Setup UI — GPU-Auswahl, Modell-Empfehlung, Docker-Script-Generator, enigma-node Dockerfile

---

## 1. Ziel

Provider können über die Web-UI `/setup` ihren LLM-Node konfigurieren. Basierend auf ihrer GPU wählen sie zwischen einem großen Modell (Tiefe) oder mehreren kleinen Modellen (Breite). Das System generiert ein Docker Compose Script + Installationsscript das alles automatisch aufbaut und beim enigma-server registriert.

---

## 2. Neue Komponenten

| Komponente | Beschreibung |
|---|---|

| `web/lib/gpu-tiers.ts` | GPU-Tier-Definitionen und VRAM-Berechnungen |

## 3. Enigma-Node Dockerfile

`enigma/Dockerfile` — Multi-stage Go Build:

```dockerfile
FROM golang:1.25-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -o enigma-node ./cmd/node

FROM alpine:3.19
RUN apk add --no-cache ca-certificates
COPY --from=builder /app/enigma-node /usr/local/bin/enigma-node
ENTRYPOINT ["enigma-node"]
```

Image-Name: `ghcr.io/enigma/enigma-node:latest` (oder `enigma/node:latest` lokal)

Provider können das Image selbst bauen (`docker build -t enigma/node .`) oder das generierte Script baut es aus dem lokalen Verzeichnis.

## 5. `/setup` Seite

### 5.1 Ablauf

1. Provider wählt GPU aus Dropdown
2. System zeigt zwei Tabs: **Tiefe** (1 großes Modell) und **Breite** (mehrere kleine)
3. Provider kann Modelle manuell per Checkbox anpassen — live VRAM-Anzeige summiert auf
4. Eingabe: Enigma-Server URL (vorbefüllt aus `NEXTAUTH_URL`)
5. Klick "Script herunterladen":
   - `docker-compose.yml` — alle Services
   - `install.sh` — Einzeiler-Installer
6. Status-Bereich: zeigt aktuell registrierte Nodes dieses Providers (aus enigma-server)

### 5.2 VRAM-Validierung

Live-Anzeige: `Gewählt: 10GB / Verfügbar: 12GB ✅`

Warnung wenn über VRAM-Limit: `⚠️ 14GB gewählt — überschreitet 12GB GPU`

### 5.3 Enigma-Server URL

Provider gibt die URL ein unter der enigma-server erreichbar ist (z.B. `http://192.168.0.100:8080`). Diese wird in docker-compose.yml und install.sh eingebettet.

## 6. Generierte Dateien

### 6.1 `docker-compose.yml`

Pro gewähltem Modell: ein Ollama-Service + ein enigma-node-Service.
Jeder Ollama-Service bekommt einen eigenen Port (11434, 11435, 11436, ...).

```yaml
# Generated by Enigma Network
# Provider: volker@beinlich.de
# GPU: RTX 4090 (24GB)
# Models: gemma3:12b, gemma3:4b

version: '3.8'

services:
  ollama-gemma3-12b:
    image: ollama/ollama
    runtime: nvidia
    environment:
      - NVIDIA_VISIBLE_DEVICES=all
    ports:
      - "11434:11434"
    volumes:
      - ollama-gemma3-12b-data:/root/.ollama
    restart: unless-stopped

  enigma-node-gemma3-12b:
    image: ghcr.io/enigma/enigma-node:latest
    command:
      - "-server"
      - "http://192.168.0.100:8080"
      - "-backend"
      - "ollama"
      - "-backend-addr"
      - "ollama-gemma3-12b:11434"
    depends_on:
      - ollama-gemma3-12b
    restart: unless-stopped

  ollama-gemma3-4b:
    image: ollama/ollama
    runtime: nvidia
    environment:
      - NVIDIA_VISIBLE_DEVICES=all
    ports:
      - "11435:11434"
    volumes:
      - ollama-gemma3-4b-data:/root/.ollama
    restart: unless-stopped

  enigma-node-gemma3-4b:
    image: ghcr.io/enigma/enigma-node:latest
    command:
      - "-server"
      - "http://192.168.0.100:8080"
      - "-backend"
      - "ollama"
      - "-backend-addr"
      - "ollama-gemma3-4b:11434"
    depends_on:
      - ollama-gemma3-4b
    restart: unless-stopped

volumes:
  ollama-gemma3-12b-data:
  ollama-gemma3-4b-data:
```

**CPU-Fallback:** Kein `runtime: nvidia`, kein `NVIDIA_VISIBLE_DEVICES`.

### 6.2 `install.sh`

```bash
#!/bin/bash
set -e

echo "=== Enigma Node Installer ==="
echo "GPU: RTX 4090 | Models: gemma3:12b, gemma3:4b"

# Docker prüfen / installieren
if ! command -v docker &> /dev/null; then
  curl -fsSL https://get.docker.com | sh
fi

# NVIDIA Container Toolkit (wenn GPU vorhanden)
if command -v nvidia-smi &> /dev/null; then
  distribution=$(. /etc/os-release; echo $ID$VERSION_ID)
  curl -s -L https://nvidia.github.io/nvidia-docker/gpgkey | sudo apt-key add -
  curl -s -L https://nvidia.github.io/nvidia-docker/$distribution/nvidia-docker.list | \
    sudo tee /etc/apt/sources.list.d/nvidia-docker.list
  sudo apt-get update && sudo apt-get install -y nvidia-container-toolkit
  sudo systemctl restart docker
fi

# Docker Compose starten
docker compose up -d

# Modelle laden (nach Ollama-Start)
echo "Warte auf Ollama..."
sleep 10
docker exec enigma-ollama-gemma3-12b-1 ollama pull gemma3:12b
docker exec enigma-ollama-gemma3-4b-1 ollama pull gemma3:4b

echo "=== Setup abgeschlossen! ==="
echo "Nodes registrieren sich automatisch bei http://192.168.0.100:8080"
```

## 7. API Routes

### `GET /api/setup/script?gpu=rtx4090&models=gemma3:12b,gemma3:4b&server=http://...`

- Auth: Session erforderlich, Rolle PROVIDER oder ADMIN
- Response: `Content-Disposition: attachment; filename="docker-compose.yml"`
- Generiert docker-compose.yml dynamisch

### `GET /api/setup/install?gpu=rtx4090&models=gemma3:12b,gemma3:4b&server=http://...`

- Auth: Session erforderlich, Rolle PROVIDER oder ADMIN
- Response: `Content-Disposition: attachment; filename="install.sh"`
- Generiert install.sh dynamisch

## 8. Node-Status auf /setup

Zeigt alle Nodes die aktuell beim enigma-server registriert sind. Da enigma-server keine User-Zuordnung kennt, wird die Liste **aller Online-Nodes** gezeigt (für PoC akzeptabel). Provider sieht so ob seine Nodes erfolgreich registriert sind.

Zukunft: Node-ID in `User.nodeId` (Array) speichern für User-spezifische Ansicht.

## 10. Out of Scope (Phase 4)

- Automatischer SSH-Zugriff auf Provider-PC
- GPU-Auto-Erkennung im Browser
- Monitoring einzelner Container
- ENI-Reward-Zuweisung pro Provider-Account
- Docker Hub / GHCR Image-Publishing (Provider baut Image lokal)
